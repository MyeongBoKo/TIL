# 객체 배열(= 참조 변수 배열)
- 객체 역시 배열로 다루는 것이 가능하다. 이때 객체 배열 안에 객체가 저장되어 있는 것이 아닌, 객체의 주소가 저장되어 있다. 즉, 객체 배열은 참조변수들을 하나로 묶은 참조 변수 배열이다. 그래서 **객체 배열** 를 **참조 변수 배열**이라고 한다.

## 객체 배열 생성 및 실행
- Code 타입의 참조변수가 3개로 이루어진 객체 배열
- 참조 변수들이 만들어진 것이지, *객체가 저장되지 않은 상태이다.*
```java
// 객체 배열 생성
Code c1, c2, c3; -> Code[] codeArr = new Code[3];
``` 
- 객체를 생성해서 배열의 각 요소에 저장한다.
  
```java
// 객체 생성 후 배열의 각 요소에 저장
codeArr[0] = new Code();
codeArr[1] = new Code();
codeArr[2] = new Code();
 ```
- 배열의 초기화를 이용해서 간단하게 한 줄로 표현한다.

 ```java
 codeArr[] = { new Code(), new Code(), new Code() };
 ```
 - 객체가 너무 많아서 직접 작성하기 어려울때, 반복문을 이용해서 표현할 수 있다.
  
  ```java
  Code[] codeArr = new Code[50];   // 배열의 길이가 50
  
  for (int i = 0; i < codeArr.length; i++>) {
    codeArr[i] = new Code();
  }
  ```

1. 설계도
2. 데이터 + 함수
3. 사용자 정의 타입

## 2. 데이터 + 함수
변수| 하나의 데이터를 저장할 수 있는 공간
--|--
배열| 같은 종류의 여러 데이터를 하나의 집합으로 저장할 수 있는 공간
구조체| 서로 관련된 여러 데이터를 종류에 관계없이 하나의 집합으로 저장할 수 있는 공간
클래스|데이터와 함수의 결합(구조체 + 함수)

- 자바의 경우 변수와 함수를 하나의 클래스에 정의해서 서로 관계가 깊은 변수와 함수들을 함께 다룰 수 있게 했다.

## 3. 사용자 정의 타입
- 자료형외에 프로그래머가 서로 관련된 변수들을 묶어서 하나의 타입으로 새로 추가한 것을 말한다.
- 자바의 경우 클래스가 곧 사용자 정의 타입이 된다.
