# 변수(Variable)
- 변수는 클래스 변수, 인스턴스 변수, 지역 변수로 이뤄졌다.
- 변수는 선언된 위치에 따라서 변수의 종류가 결정된다.
## 선언된 위치에 따른 변수의 종류

```java
class variables
{                                           // 클래스 영역 시작
    int iv;         // 인스턴스변수(iv)
    static int cv;  // 클래스변수(cv)

    void methon()
    {                                       // 메서드 영역 시작
        int lv = 0; // 지역변수(lv)
    }                                       // 메서드 영역 끝
}                                           // 클래스 영역 끝
```
변수의 종류 | 선언 위치 | 생성시기
:--|:--|:--
클래스변수(cv)| 클래스 영역 | 클래스가 메모리에 올라갈 때
인스턴스변수(iv)| 클래스 영역 |인스턴스가 생성되었을 때
지역변수(lv) | 클래스 영역 이외의 영역 | 변수 선언문이 수행되었을 때   

### `1. 인스턴스변수(instance variable)`
- 클래스 영역에서 선언된다.

- 클래스의 인스턴스 생성할 때 만들어진다.
- 인스턴스의 값을 읽기 위해서는 먼저 인스턴스를 생성해야 한다.
- 객체 인스턴스 변수의 묶음이다.
- 인스턴스는 독립적인 저장공간을 가지고 있기 때문에 서로 다른 값을 가질 수 있다. 그리고 인스턴스마다 고유한 상태를 유지해야 하는 속성의 경우,인스턴스변수로 선언해야 한다.
  
### `2. 클래스변수(class variable)`
- 인스턴스 변수 앞에 static를 붙이면 된다.

- 클래스 변수는 모든 인스턴스가 공통된 저장공간을 공유한다.
- 한 클래스의 모든 인스턴스들이 공통적인 값을 유지해야 하는 속성의 경우, 클래스 변수로 선언해야 한다.
- 객체를 생성하지 않고 바로 사용할 수 있다. 
- 자동생성된다.

### `3. 지역변수`
- 메서드 내에서 선언되고, 메서드 내에서만 사용 가능하다.

- 메서드가 종료되면 사라져서 사용할 수 없게 된다.
- 매서드의 영역, 즉 {}안에서만 작동하기 때문에 이름은 같지만 서로 같은 변수는 아니다.
  
## 클래스변수와 인스턴스변수
```java
class Apartment{
    String brand;               // 개별적인 변수는 iv
    int floor;                  

    static int width = 100;     // 공통적인 변수는 cv
    staitc int height = 200;    
}

```
- **인스턴스변수는 인스턴스가 생성될 때 마다 생성되기 때문에 인스턴스마다 각각의 다른 값을 가지고 있지만, 클래스변수는 모든 인스턴스가 하나의 저장공간을 공유하기 때문에 항상 공통된 값을 갖는다.**

# Method
## 메서드란
- 특정 작업을 수행하는 일련의 문장들을 하나로 묶은 것이다.

- 값(입력)을 받아서 처리하고, 결과를
   반환(출력)한다.
- 클래스에 독립적이다.

## 메서드의 장점
- 코드의 중복을 줄일 수 있다.

- 코드의 관리가 쉽다.
- 코드를 재사용할 수 있다.
- 코드가 간결하기 때문에 이해하기 쉽다.

## 메서드의 선언과 구현
```
반환타입 메서드이름 (타입 변수명, 타입 변수명,  ... )             - 선언부
```
```java
{                                                               - 구현부
    // 메서드 호출시 수행될 코드
}
```
```java
int add(int a, int b)                                           - 선언부
```
```java
{
    int result = a + b;                                         - 구현부
    return result;      // 호출한 메서드로 결과를 반환한다.
}
```

### 1. 메서드 선언부
- 메서드의 이름, 매개변수 선언(입력), 반환타입(출력)으로 구성  
  `int-(반환타입)` `add -(메서드의 이름)` `(int x, int y)-(매개변수 선언)` { ... }

- 입력은 0 ~ n개. 즉, 입력의 경우 없어도 되고 여러개도 가능하다. 
- 출력은 0 ~ 1개. 만약 작업 결과가 여러개이면 배열 또는 객체를 이용해서 나타내야 한다.

### 2. 매개변수 선언
- 메서드가 작업을 수행하는데 필요한 값(입력)을 제공받기 위함이다.

- 변수의 타입이 같아도 매개변수에서는 변수의 타입을 생략할 수 없다.
- 매개변수의 개수 제한없고, 만약 입력해야 하는 값이 많으면 배열 또는 참조변수를 사용한다.
- 값을 받지 않는다면 빈칸()으로 표현한다.

### 3. 메서드의 이름
- 동사인 경우로 이름을 작성한다.
- 메서드의 기능을 쉽게 알 수 있게 작성해야 한다.

### 4. 반환타입 
- 작업수행 결과인 반환값의 타입을 적는다. 이때 반환값이 없으면 *void*를 사용한다.

### 5. 메서드의 구현부
- 메서드를 호출했을 때 수행될 문장들을 넣는다.

### 6. return 문
- return 문은 작업을 수행한 결과인 반환값을 호출한 메서드로 전달하는 역할을 한다.
  
- 반환 타입이 void가 아니면, 반드시 구현문 안에 return 문이 있어야 한다.
- 작업을 수행한 결과인 반환값을 호출한 메서드로 전달할 때, 이 값의 타입과 반환타입과 일치하거나 자동 형변환이 가능해야 한다.
- 입력받은 매개변수가 여러 개여도 출력은 하나가 최대이다.
- 조건문을 이용할 때 참인 경우만 존재하면 거짓의 경우 return이 없기 때문에 에러가 발생한다.
- 반환 타입이 void면 void는 생략 가능하다. 컴파일러가 자동으로 추가하기 때문이다.
```java
void printCar(int x) {
    // 수행되는 문장
//    return; // 반환 타입이 void이기 때문에 생략가능. 컴파일러가 자동으로 추가
}
```
```java
// 잘못 작성한 코드
int max(int a, int b) {
    if (a > b)
        return a;   // 참일 때만 실행
}

// 옳게 작성한 코드
int max(int a, int b) {
    if (a > b)
        return a;
    else
        return b;
}
```

### 7. 지역변수
- 메서드 내에 선언된 변수를 말한다.
- 매개변수도 메서드 내에 선언된 것으로 간주하여 지역변수이다.


## 메서드의 호출

<메서드를 호출하는 방법>
```java
메서드 이름(값1, 값2, ...);

printcar();             // void printcar() 을 호출
int result = add(3,5);  // int add(int x, int y) 를 호출하고, 결과를 result에 저장
```

### 인자(argument)와 매개변수(parameter)
- 메서드를 호출할 때 괄호() 안에 지정해준 값들을 '인자' 또는 '인수'라고 한다.

- 인자의 개수와 순서는 호출된 메서드에 선언된 매개변수와 일치해야 한다.
- 인자는 메서드가 호출되면서 매개변수에 대입힌다. 
- 이때 타입은 매개변수의 타입과 일치하거나 자동 형변환이 가능해야 한다.

### 참고
- 같은 클래스 내의 메서드끼리 참조변수를 사용하지 않고 서로 호출 가능하다.
- static메서드는 같은 클래스 내의 인스턴스 메서드를 호출할 수 없다.

### 메서드의 실행 흐름
```java
MyMath mm = new MyMath();

long value = mm.add(1L, 2L);

long add(Long a, Long b) {
    long result = a + b;
    return result;
}
```
1. mm이라는 참조변수 생성 후 MyMath클래스의 인스턴스를 생성시킨 후 연결시킨다.
2. main 메서드에서 메서드 add를 호출한다. 이때 인수 1L와 2L는 add 메서드의 매개변수에 대입된다.
3. 메서드 add의 문장이 실행된다.
4. return문을 만난 후에 호출한 메서드로 값이 반환된다. 이때 메서드는 호출시 넘겨받은 값으로 연산을 수행하고, 그 결과를 반환하면서 종료된다. 반환된 값은 대입연산자에 의해 대입된다.
5. 호출한 메서드로 되돌아 온 후에 이후의 문장들이 실행된다.
